💸 02 — Sistema de Gorjetas (PIX)

Fluxo completo de gorjetas via PIX, desde o até a confirmação.

Arquivo Principal

src/pages/artist/ArtistTip.jsx — Página pública onde o fã envia o pedido + gorjeta.

Fluxo do Usuário

  1. Fan acessa a página do artista via link público (/artist/:artistId/tip)
  2. Preenche o formulário: nome, música desejada (opcional), mensagem
  3. Escolhe o valor da gorjeta: R$ 5, R$ 10, R$ 20, R$ 30, R$ 50 ou personalizado
  4. Clica em "Adicionar Gorjeta" (envia pedido + gorjeta) ou "Fazer Pedido sem Gorjeta"
  5. Sistema gera QR Code PIX dinâmico com o valor embutido + código copia-e-cola
  6. Fan paga o PIX (app do banco)
  7. Frontend faz polling na tabela pending_tips a cada 3 segundos
  8. Quando o webhook do Asaas confirma o pagamento → status atualiza para "confirmed"
  9. Tela de agradecimento com estrelas para avaliar o artista

Detalhes Técnicos

Edge Function: asaas-create-pix

  • Endpoint: POST /asaas-create-pix
  • Modo gorjeta: Cria cliente Asaas (com CPF aleatório válido), cria pagamento via /v3/payments com billingType: 'PIX' e split (70/30)
  • Resposta: Retorna paymentId, pixPayload (copia-e-cola), pixQrCode (base64)
  • Salva registro em pending_tips com status "pending"
  • Cria registro em music_requests para o pedido musical

Tabela pending_tips

Coluna Tipo Descrição
id UUID ID único
user_id UUID Fan que enviou a gorjeta
artist_id UUID Artista destinatário
amount DECIMAL Valor da gorjeta em reais
status TEXT pending → confirmed → transferred
asaas_transaction_id TEXT ID do pagamento no Asaas
music_request_id UUID Pedido musical vinculado

Etapas da Tela (STAGE)

Estágio Constante Descrição
FORM STAGE.FORM = 0 Formulário de preenchimento
ORDER_ONLY_THANKS STAGE.ORDER_ONLY_THANKS = 1 Obrigado sem gorjeta
TIP_VALUE STAGE.TIP_VALUE = 2 Escolha do valor
PIX_PAYMENT STAGE.PIX_PAYMENT = 3 QR Code PIX + polling
FINAL_THANKS STAGE.FINAL_THANKS = 4 Agradecimento + avaliação

👑 03 — Assinatura Pro

Plano de assinatura mensal para artistas com benefícios exclusivos.

Recursos do Plano Pro

  • 💰 100% das gorjetas (sem taxa da plataforma)
  • ✨ Badge Pro (coroa dourada) no perfil e nas buscas
  • 📊 Artistas Pro aparecem primeiro nas buscas

Preço

R$ 49,90/mês — pagamento via PIX com QR Code.

Fluxo de Assinatura

  1. Artista clica no botão "Ser Pro" (ícone Crown) no header
  2. Abre modal com lista de benefícios + preço + botão "Assinar Agora"
  3. Artista informa CPF/CNPJ para cadastro no Asaas
  4. Edge function asaas-create-pix (mode: 'subscription') cria a assinatura recorrente
  5. Retorna QR Code do primeiro pagamento
  6. Artista paga → webhook PAYMENT_RECEIVED ativa o is_pro = true
  7. Frontend também faz polling a cada 5s → auto-ativa ao detectar pagamento aprovado

Detalhes Técnicos

Ciclo de Vida da Assinatura

  • Criação: POST /v3/subscriptions via Asaas API → salva asaas_subscription_id + pro_expires_at (now + 30 dias)
  • Ativação: Via webhook PAYMENT_RECEIVED ou polling do frontend
  • Expiração: AuthContext.jsx verifica no login se pro_expires_at < now → is_pro = false
  • Revogação via webhook: Evento SUBSCRIPTION_INACTIVATED também reverte is_pro

Arquivos Envolvidos

Arquivo Função

asaas-create-pix/index.ts Cria assinatura Asaas, retorna primeiro pagamento PIX

🏪 04 — Marketplace Asaas (Subcontas)

Sistema de criação automática de subcontas Asaas para artistas.

O que é

Em vez de o artista manualmente inserir o ID da carteira Asaas, o plataforma cria uma subconta automaticamente via API do Asaas. O artista recebe um e-mail de ativação do Asaas.

Fluxo

  1. Artista acessa Perfil → "Conectar Conta Asaas"
  2. Informa CPF/CNPJ
  3. Edge function asaas-create-subaccount chama POST /v3/accounts
  4. Retorna walletId + apiKey → salvos no banco
  5. UI mostra: "Conta Criada — Aguardando Ativação"
  6. Artista recebe e-mail do Asaas e ativa a conta

Detalhes Técnicos

Edge Function: asaas-create-subaccount

  • Requer: cpfCnpj no body da requisição
  • Cria: Subconta via POST /v3/accounts com dados do artista
  • Retorna: walletId (wal_xxx) + apiKey
  • Salva: asaas_wallet_id, asaas_api_key, asaas_account_status = 'pending'
  • ⚠️ Restrição: Apenas contas CNPJ podem criar subcontas no Asaas

Colunas Adicionadas ao Banco

Coluna Tabela Descrição
asaas_wallet_id artists ID da carteira Asaas (wal_xxx)
asaas_api_key artists Chave de API da subconta
asaas_account_status artists Status: pending / active
pix_key artists Chave PIX para fallback
cpf_cnpj artists CPF/CNPJ do artista

🔔 05 — Sistema de Notificações

Notificações em tempo real via Supabase Realtime.

Funcionamento

  • Realtime: Supabase Realtime escuta a tabela notifications para o user_id logado
  • Badge: Contador de não-lidas no ícone 🔔 no header
  • Sound: Som de notificação ao receber novos pedidos (ArtistRequests.jsx)
  • Limpar: Botão para deletar todas as notificações

Implementação no Frontend

AppLayout.jsx

  • useEffect busca notificações iniciais (últimas 20)
  • Inscreve-se no canal notifications-realtime
  • Escuta eventos INSERT e UPDATE filtrados por user_id
  • Notificações são adicionadas em tempo real ao estado local
  • Badge mostra unreadCount = notificações sem read: true

ArtistRequests.jsx — Som de Notificação

  • Quando um novo pedido pendente é detectado no polling → toca som
  • Som em Base64 (WAV), volume 30%
  • Compara IDs anteriores vs atuais para detectar novos pedidos

🔗 06 — Webhook Asaas

Endpoint que recebe e processa eventos do Asaas automaticamente.

Detalhes da Edge Function

asaas-webhook/index.ts

  • Autenticação: Token via header asaas-access-token ou query string ?token=
  • Health Check: GET / retorna {"status": "alive"}
  • Webhook ID: 1b8aa9b9-a29e-4b3a-ba60-f69c10dee1ac
  • Status: enabled, não interrompido, 12 eventos configurados

Eventos Processados

Evento Ação
PAYMENT_RECEIVED / PIX_RECEIVED 1. Verifica se é pagamento de assinatura (ativa Pro)
2. Busca pending_tips por asaas_transaction_id
3. Fallback: busca por valor (FIFO)
4. Atualiza status para "confirmed"
5. Atualiza rating do artista
6. Cria notificação ao artista
SUBSCRIPTION_INACTIVATED Busca artista por asaas_subscription_id → reverte is_pro = false
SUBSCRIPTION_REACTIVATED Ativa novamente is_pro = true
CHARGE_RECEIVED Processa cobrança recorrente (mesmo fluxo de pagamento)

Transferência Automática (Split)

Após confirmar o pagamento, o webhook transfere a parte do artista:

  • 1ª tentativa: Transferência wallet-to-wallet via /v3/transfers com walletId
  • 2ª tentativa (fallback): Transferência PIX para a pix_key do artista
  • Se falhar: Registra como failed para reconciliação manual

⚠️ Nota: Transferências wallet-to-wallet do Asaas requerem contas vinculadas (mesma raiz/subconta).

⚖️ 07 — Split de Pagamentos

Divisão automática entre artista e plataforma.

Proporção

Destinatário Porcentagem Exemplo (R$ 10,00)
Artista 70% R$ 7,00
TocaMais (Plataforma) 30% R$ 3,00

Como funciona

  1. No momento da criação do pagamento PIX, o asaas-create-pix envia o parâmetro split ao Asaas
  2. O split é configurado no próprio pagamento — não precisa de transferência separada
  3. Webhook como backup: Se o artista tem asaas_wallet_id, o webhook também tenta transferir via /v3/transfers

Casos Especiais

Artista Pro (100%)

Artistas com is_pro = true recebem 100% da gorjeta (sem taxa da plataforma). O split não é aplicado nesse caso.

Sem Conta Asaas

Se o artista não tem asaas_wallet_id, o valor fica retido na conta da plataforma até que o artista conecte sua carteira.

🔐 08 — Página de Login e Reset de Senha

Autenticação completa com recuperação de senha.

Tela de Login

  • Modo Login (email + senha) e Cadastro (nome + telefone + email + senha + tipo de perfil)
  • 3 botões de login rápido (Demo): Artista / Casa Show / Particular
  • Seleção de perfil no cadastro com ícones visuais
  • Validação de campos obrigatórios

Reset de Senha

  1. Link "Esqueci minha senha" abaixo do campo de senha (apenas no modo login)
  2. Clique abre tela com campo de e-mail (pré-preenchido)
  3. Chama supabase.auth.resetPasswordForEmail(email, { redirectTo })
  4. Tela de confirmação: "E-mail enviado!" com ícone verde ✅
  5. Botão "← Voltar ao login"

3 Visões do Componente

View Estado Descrição
'login' Padrão Login ou cadastro com formulário
'forgot' Esqueci senha Campo de e-mail + enviar link
'forgot_sent' Enviado Confirmação de envio do e-mail

Detalhes Técnicos

  • Campos: view ('login' | 'forgot' | 'forgot_sent'), resetEmail
  • Importações: KeyRound, CheckCircle do lucide-react
  • handleForgotPassword: valida e-mail, chama Supabase, alterna view
  • Redireciona para /login via redirectTo
